# Tasks

The active-intent subset of the [backlog](backlog.md) — the short list of what we actually mean to do next, in priority order. Where `backlog.md` is the passive, trigger-gated superset of everything deferred, `tasks.md` is what we've decided to pull forward.

## How this file works

- An item lands here when we decide to actively do it — promoted from `backlog.md` (a trigger fired, or we chose to prioritize it) or added net-new.
- **Format:** `- [ ] **Title** — one sentence. ↳ [source](...)` — check the box when done, then **delete** it (the docs become the source of truth). Tasks don't linger here as ✅.
- Keep it short and honest: if nothing is actively planned, this file is nearly empty — that's the correct state, not a gap to fill.
