# Backlog

The passive, trigger-gated superset of everything **deferred, punted, or left uncertain** in this project — kept so nothing is lost when scope moves on. Most entries resurface only when a stated condition fires; this is not a to-do list (see [tasks.md](tasks.md) for what we mean to do next).

## How this file works

- **Backlog vs [tasks](tasks.md):** backlog is the deferred *superset* (mostly trigger-gated); `tasks.md` is the active-intent *subset*. An item moves **backlog → tasks** when we decide to pull it forward.
- **Lifecycle:** an item is **deleted** from this file when it ships (the docs become the source of truth) or when it's promoted into `tasks.md`. Don't leave shipped items here.
- **Format:** `- **Title** — one sentence. _Trigger:_ condition. ↳ [source](...)`, grouped under `## Area` headings.
